# Mock 演示模式

Mock 是 Idea Bubble 的默认产品模式，不是静态占位页。首次打开、清除本地 AI 配置或主动选择 Mock 时，不需要 API Key、环境变量或外部模型费用，也可以走完整工作流。

## 行为

- 同一输入和发散方向得到稳定结果，便于测试与演示；
- 每次文字发散严格生成 10 个不重复气泡；
- “蜂巢”等验收输入返回语义相关而非随机占位词；
- 总结、计划和生图提示词遵循与真实 Provider 相同的 Zod Schema；
- Mock 图片分析只使用文件名和 MIME 生成演示数据，并明确声明没有理解真实像素；
- 视频/GIF 保持 `unsupported`，不会伪装成已经理解内容；
- 项目保存、刷新恢复、版本、导出和移动端布局与真实模式一致。

Mock 不调用外部 AI Provider，也不需要读取或上传 BYOK 配置。项目与导出仍遵守各自已有的数据流；“Mock”不等同于浏览器安装型离线 PWA。

## 切换规则

- 干净浏览器始终以 Mock 启动；Vercel 环境变量不能替用户切换到真实 Provider。
- 保存完整的本地 BYOK 配置后，用户可以主动使用真实 Provider。
- 真实 Provider 缺少 API Key、当前任务模型或有效 HTTPS Base URL 时，界面应提示补全配置，不会借用部署方 Key 或写死模型。
- 点击“清除本地配置”会删除当前 Origin 的设置并恢复 Mock，但不会删除项目。
- 项目备份或恢复不会携带 Provider 选择和密钥；在另一浏览器恢复项目后仍使用该浏览器自己的模式。

## 用途

- 首次体验和无 Key 演示；
- UI、画布与工作流开发；
- 单元测试和 Playwright 端到端测试；
- 验证文档导出而不产生模型费用；
- Provider CORS 不支持浏览器直连时的安全降级选择。

Mock 不应用于验证真实模型的指令遵循、图像识别质量、延迟、费用或 CORS。接入真实 Provider 后仍需用用户指定的模型验证结构化输出和视觉能力；测试 Key 只保存在测试 BrowserContext 中，不能写进仓库、Vercel 或测试报告。
