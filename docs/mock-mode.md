# Mock 演示模式

Mock 是完整的离线产品模式，不是静态占位页。设置 `AI_PROVIDER=mock`，或在真实 Provider 缺少 API Key/任务模型时，AI 任务会自动使用本地确定性生成器。

## 行为

- 同一输入和发散方向得到稳定结果，便于测试与演示。
- 每次文字发散严格生成 10 个不重复气泡。
- 总结、计划和生图提示词都遵循与真实 Provider 相同的 Zod Schema。
- Mock 图片分析只使用文件名和 MIME 生成演示数据，并在结果中明确声明没有读取真实像素内容。
- 视频/GIF 不会被 Mock 成“已理解”；它们保持 `unsupported`。

## 用途

- 首次安装和无网络演示；
- UI/工作流开发；
- 单元测试与 Playwright 端到端测试；
- 验证文档导出，不产生模型费用。

Mock 不应用于验证真实模型的指令遵循、图像识别质量或延迟。接入真实 Provider 后仍需用目标模型跑结构化输出和视觉能力验收。
